# Bad Commit Message Examples (Anti-Patterns)

## ‚ùå No Category Prefix

Fix memory leak

‚úÖ Should be:
LibJS: Fix memory leak in running_execution_context()

---

## ‚ùå Non-Imperative Mood

LibWeb: Added CSS grid support

‚úÖ Should be:
LibWeb: Add CSS grid support

---

## ‚ùå Past Tense

Sentinel: Fixed fingerprinting detector crash

‚úÖ Should be:
Sentinel: Fix fingerprinting detector crash

---

## ‚ùå Gerund Form (-ing)

LibWeb: Adding canvas API support

‚úÖ Should be:
LibWeb: Add canvas API support

---

## ‚ùå Too Vague

Fix bug

‚úÖ Should be:
LibJS: Fix off-by-one error in array iteration

---

## ‚ùå First Line Too Long

LibWeb: Implement the complete constraint validation API for HTML input elements including all validity states and error messages

‚úÖ Should be (72 chars max):
LibWeb: Implement HTMLInputElement constraint validation API

---

## ‚ùå Period at End of First Line

LibWeb: Add CSS grid support.

‚úÖ Should be:
LibWeb: Add CSS grid support

---

## ‚ùå Lowercase First Letter

LibWeb: add CSS grid support

‚úÖ Should be:
LibWeb: Add CSS grid support

---

## ‚ùå No Description

fix

‚úÖ Should be:
LibWeb: Fix crash when parsing invalid CSS

---

## ‚ùå Wrong Category

HTML: Add CSS grid support

‚úÖ Should be (it's in LibWeb, not just HTML):
LibWeb: Add CSS grid support

---

## ‚ùå Multiple Changes in One Commit

LibWeb: Add grid support, fix flexbox bug, update documentation

‚úÖ Should be (separate commits):
Commit 1: LibWeb: Add CSS grid support
Commit 2: LibWeb: Fix flexbox alignment bug
Commit 3: docs: Update CSS implementation status

---

## ‚ùå Explaining WHAT Instead of WHY

LibWeb: Changed variable name from x to element_count

‚úÖ Should be:
LibWeb: Use descriptive variable names in layout algorithm

Variable names like 'x' and 'i' were making the complex
layout algorithm difficult to understand. Renamed to
semantic names matching the spec terminology.

---

## ‚ùå No Spec Link for Web Standards

LibWeb: Implement constraint validation

‚úÖ Should be:
LibWeb: Implement HTMLInputElement constraint validation

Implements the constraint validation API from HTML spec:
https://html.spec.whatwg.org/multipage/form-control-infrastructure.html

---

## ‚ùå Commit Message in Body

git commit -m "LibWeb: Add canvas API" -m "This commit adds canvas API"

‚úÖ Should be:
LibWeb: Add canvas API

Implements CanvasRenderingContext2D from HTML spec.
Includes all basic drawing operations.

---

## ‚ùå Personal Attribution

LibWeb: Add CSS grid (by John Doe)

‚úÖ Should be (git tracks authorship automatically):
LibWeb: Add CSS grid support

---

## ‚ùå Emoji in Commit Message

‚ú® LibWeb: Add CSS grid support üéâ

‚úÖ Should be (professional commits only):
LibWeb: Add CSS grid support

---

## ‚ùå Merge Commit Without Context

Merge branch 'feature/my-feature'

‚úÖ Should be:
Merge feature/css-grid-implementation

Implements CSS grid layout engine with full spec compliance.
Includes layout algorithms, computed values, and painting.

---

## ‚ùå "Fix" Without Saying What Was Broken

Fix it

‚úÖ Should be:
LibJS: Fix use-after-free in garbage collector

The GC was accessing freed memory during weak reference cleanup.

---

## ‚ùå WIP Commit in Main Branch

WIP: working on canvas

‚úÖ Should be (never push WIP to main):
LibWeb: Add initial canvas API implementation

---

## ‚ùå Debug/Temp Commit Messages

asdf
test
debug
temp
aaa

‚úÖ Should be:
Never commit with debug messages. Use proper descriptions.

---

## ‚ùå Inconsistent Category Names

libweb: Add CSS grid   (lowercase)
Libweb: Add flexbox    (wrong case)

‚úÖ Should be (consistent CamelCase):
LibWeb: Add CSS grid support
LibWeb: Add flexbox layout support

---

## ‚ùå Generic "Update" Without Details

Update file

‚úÖ Should be:
LibWeb: Implement CSS cascade algorithm

Updated StyleResolver to properly handle cascade layers
and @import order per CSS Cascade spec.

---

## ‚ùå Combining Unrelated Changes

LibWeb: Add canvas API and fix unrelated memory leak in parser

‚úÖ Should be (separate commits):
Commit 1: LibWeb: Add canvas API
Commit 2: LibWeb: Fix memory leak in CSS parser

---

## ‚ùå Referencing Code Lines

Fix bug on line 42

‚úÖ Should be:
LibJS: Fix array bounds check in element access

The bounds check was comparing unsigned values incorrectly,
causing out-of-bounds access for negative indices.

---

## ‚ùå Missing Context for "Fix"

Fix crash

‚úÖ Should be:
LibWeb: Fix crash when rendering empty flex containers

The flex layout algorithm assumed at least one child,
causing null pointer dereference for empty containers.

---

## ‚ùå Abbreviations Without Explanation

LibWeb: Add GC for DOM nodes

‚úÖ Should be (spell out on first use):
LibWeb: Add garbage collection for DOM nodes

Implements generational garbage collector to automatically
reclaim unused DOM nodes and prevent memory leaks.

---

## ‚ùå Code Review Comments as Commit Messages

Address review feedback
Fix nits
Apply suggestions

‚úÖ Should be (squash into original commit):
Use git commit --amend or git rebase -i to incorporate
feedback into the original commit with proper message.

---

## ‚ùå Version/Date in Commit Message

LibWeb: Add CSS grid (v1.2.3, 2024-11-01)

‚úÖ Should be (git tracks dates/versions):
LibWeb: Add CSS grid support

---

## ‚ùå External References Without Context

See #123

‚úÖ Should be:
LibWeb: Fix input validation bug

The email input type was not properly validating TLDs.

Fixes #123
